A versatile and accessible input component that supports multiple input types, validation states, icons, labels, and custom styling. This component is designed to handle various form scenarios including text inputs, passwords, emails, numbers, and more. Perfect for building robust forms with consistent styling and behavior across your application.

## Features

- **Multiple Input Types**: Text, password, email, number, tel, url, search, and more
- **Validation States**: Error, success, and warning states with custom messages
- **Icon Support**: Leading and trailing icons with click handlers
- **Label Positioning**: Top, floating, or inline label options
- **Help Text**: Additional context or instructions below the input
- **Size Variants**: Small, medium, and large sizes
- **Disabled State**: Proper disabled styling and behavior
- **Custom Styling**: Full control with Tailwind CSS classes
- **Accessibility**: ARIA attributes, proper labeling, and keyboard navigation

## Props

| Prop             | Type                   | Default     | Description                                           |
| ---------------- | ---------------------- | ----------- | ----------------------------------------------------- |
| `type`           | `string`               | `'text'`    | HTML input type (text, email, password, number, etc.) |
| `label`          | `string`               | `undefined` | Label text for the input                              |
| `placeholder`    | `string`               | `''`        | Placeholder text                                      |
| `value`          | `string \| number`     | `''`        | Input value                                           |
| `onChange`       | `function`             | `required`  | Function called when input value changes              |
| `error`          | `string`               | `undefined` | Error message to display                              |
| `helperText`     | `string`               | `undefined` | Helper text below the input                           |
| `disabled`       | `boolean`              | `false`     | Whether the input is disabled                         |
| `required`       | `boolean`              | `false`     | Whether the input is required                         |
| `size`           | `'sm' \| 'md' \| 'lg'` | `'md'`      | Size variant of the input                             |
| `leadingIcon`    | `ReactNode`            | `undefined` | Icon to display before the input text                 |
| `trailingIcon`   | `ReactNode`            | `undefined` | Icon to display after the input text                  |
| `onIconClick`    | `function`             | `undefined` | Function called when trailing icon is clicked         |
| `className`      | `string`               | `''`        | Additional CSS classes                                |
| `inputClassName` | `string`               | `''`        | Additional CSS classes for the input element          |

### Input with Validation

```jsx
function ValidationExample() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [confirmPassword, setConfirmPassword] = useState("");

  const validateEmail = (email) => {
    const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return re.test(email);
  };

  const emailError =
    email && !validateEmail(email)
      ? "Please enter a valid email address"
      : undefined;
  const passwordError =
    password && password.length < 8
      ? "Password must be at least 8 characters"
      : undefined;
  const confirmError =
    confirmPassword && password !== confirmPassword
      ? "Passwords do not match"
      : undefined;

  return (
    <div className="space-y-4 max-w-md">
      <Input
        type="email"
        label="Email"
        placeholder="Enter your email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        error={emailError}
        required
      />

      <Input
        type="password"
        label="Password"
        placeholder="Enter your password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        error={passwordError}
        helperText="Must be at least 8 characters long"
        required
      />

      <Input
        type="password"
        label="Confirm Password"
        placeholder="Confirm your password"
        value={confirmPassword}
        onChange={(e) => setConfirmPassword(e.target.value)}
        error={confirmError}
        required
      />
    </div>
  );
}
```

### Form with Dynamic Validation

```jsx
function DynamicValidationForm() {
  const [formData, setFormData] = useState({
    firstName: "",
    lastName: "",
    email: "",
    phone: "",
    website: "",
  });

  const [errors, setErrors] = useState({});
  const [touched, setTouched] = useState({});

  const validationRules = {
    firstName: { required: true, minLength: 2 },
    lastName: { required: true, minLength: 2 },
    email: { required: true, pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/ },
    phone: { required: true, pattern: /^\+?[\d\s-()]+$/ },
    website: { pattern: /^https?:\/\/.+/ },
  };

  const validateField = (name, value) => {
    const rules = validationRules[name];
    if (!rules) return "";

    if (rules.required && !value.trim()) {
      return `${name.charAt(0).toUpperCase() + name.slice(1)} is required`;
    }

    if (rules.minLength && value.length < rules.minLength) {
      return `${
        name.charAt(0).toUpperCase() + name.slice(1)
      } must be at least ${rules.minLength} characters`;
    }

    if (rules.pattern && !rules.pattern.test(value)) {
      if (name === "email") return "Please enter a valid email address";
      if (name === "phone") return "Please enter a valid phone number";
      if (name === "website")
        return "Please enter a valid URL (http:// or https://)";
    }

    return "";
  };

  const handleChange = (name, value) => {
    setFormData((prev) => ({ ...prev, [name]: value }));

    if (touched[name]) {
      const error = validateField(name, value);
      setErrors((prev) => ({ ...prev, [name]: error }));
    }
  };

  const handleBlur = (name) => {
    setTouched((prev) => ({ ...prev, [name]: true }));
    const error = validateField(name, formData[name]);
    setErrors((prev) => ({ ...prev, [name]: error }));
  };

  return (
    <div className="max-w-md space-y-4">
      <div className="grid grid-cols-2 gap-4">
        <Input
          label="First Name"
          placeholder="John"
          value={formData.firstName}
          onChange={(e) => handleChange("firstName", e.target.value)}
          onBlur={() => handleBlur("firstName")}
          error={errors.firstName}
          required
        />

        <Input
          label="Last Name"
          placeholder="Doe"
          value={formData.lastName}
          onChange={(e) => handleChange("lastName", e.target.value)}
          onBlur={() => handleBlur("lastName")}
          error={errors.lastName}
          required
        />
      </div>

      <Input
        type="email"
        label="Email"
        placeholder="john@example.com"
        value={formData.email}
        onChange={(e) => handleChange("email", e.target.value)}
        onBlur={() => handleBlur("email")}
        error={errors.email}
        leadingIcon={<Mail className="w-4 h-4 text-gray-400" />}
        required
      />

      <Input
        type="tel"
        label="Phone"
        placeholder="+1 (555) 123-4567"
        value={formData.phone}
        onChange={(e) => handleChange("phone", e.target.value)}
        onBlur={() => handleBlur("phone")}
        error={errors.phone}
        leadingIcon={<Phone className="w-4 h-4 text-gray-400" />}
        required
      />

      <Input
        type="url"
        label="Website"
        placeholder="https://example.com"
        value={formData.website}
        onChange={(e) => handleChange("website", e.target.value)}
        onBlur={() => handleBlur("website")}
        error={errors.website}
        helperText="Optional - include http:// or https://"
      />
    </div>
  );
}
```

### Search Input with Suggestions

```jsx
function SearchWithSuggestions() {
  const [query, setQuery] = useState("");
  const [suggestions, setSuggestions] = useState([]);
  const [showSuggestions, setShowSuggestions] = useState(false);

  const mockSuggestions = [
    "React hooks",
    "React components",
    "React router",
    "JavaScript ES6",
    "JavaScript arrays",
    "JavaScript promises",
    "Node.js express",
    "Node.js mongodb",
    "Node.js authentication",
    "MongoDB queries",
    "MongoDB aggregation",
    "MongoDB indexing",
  ];

  const handleSearch = (value) => {
    setQuery(value);

    if (value.length > 1) {
      const filtered = mockSuggestions
        .filter((item) => item.toLowerCase().includes(value.toLowerCase()))
        .slice(0, 5);
      setSuggestions(filtered);
      setShowSuggestions(true);
    } else {
      setShowSuggestions(false);
    }
  };

  const handleSuggestionClick = (suggestion) => {
    setQuery(suggestion);
    setShowSuggestions(false);
  };

  const clearSearch = () => {
    setQuery("");
    setShowSuggestions(false);
  };

  return (
    <div className="relative max-w-md">
      <Input
        type="search"
        placeholder="Search documentation..."
        value={query}
        onChange={(e) => handleSearch(e.target.value)}
        onFocus={() => query.length > 1 && setShowSuggestions(true)}
        leadingIcon={<Search className="w-4 h-4 text-gray-400" />}
        trailingIcon={
          query ? <X className="w-4 h-4 text-gray-400" /> : undefined
        }
        onIconClick={query ? clearSearch : undefined}
      />

      {showSuggestions && suggestions.length > 0 && (
        <div className="absolute top-full left-0 right-0 bg-white border border-gray-200 rounded-md shadow-lg mt-1 z-10">
          {suggestions.map((suggestion, index) => (
            <button
              key={index}
              className="w-full px-3 py-2 text-left hover:bg-gray-100 first:rounded-t-md last:rounded-b-md"
              onClick={() => handleSuggestionClick(suggestion)}
            >
              {suggestion}
            </button>
          ))}
        </div>
      )}
    </div>
  );
}
```

## Accessibility Features

- **Proper Labeling**: Labels are correctly associated with inputs using `htmlFor`
- **ARIA Attributes**: Includes `aria-invalid`, `aria-describedby` for error states
- **Error Announcements**: Screen readers announce validation errors
- **Keyboard Navigation**: Full keyboard support with proper focus management
- **High Contrast**: Works well with high contrast modes and themes

## Best Practices

1. **Always provide labels** - Use clear, descriptive labels for all inputs
2. **Use appropriate input types** - Choose the right HTML input type for better UX
3. **Provide helpful error messages** - Make errors specific and actionable
4. **Use placeholder text wisely** - Placeholders should be examples, not instructions
5. **Group related inputs** - Use fieldsets and proper spacing for form organization
6. **Test with screen readers** - Ensure all form elements are properly announced
7. **Consider mobile users** - Ensure inputs are appropriately sized for touch interaction
8. **Validate on appropriate events** - Balance between immediate feedback and user interruption
