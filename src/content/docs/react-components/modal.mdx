The following modal component is a versatile overlay dialog that displays content above the main application interface. This modal is highly customizable with Tailwind CSS and includes features like backdrop click to close, custom titles, content areas, and footer sections. Perfect for displaying forms, confirmations, or any content that requires focused user attention.

## Features

- **Backdrop Click to Close**: Click outside the modal to close it
- **Customizable Styling**: Full control over modal and overlay appearance with Tailwind CSS
- **Flexible Content**: Support for title, main content, and footer sections
- **Close Button**: Built-in close button with hover effects
- **Responsive Design**: Works seamlessly across different screen sizes
- **Accessible**: Proper focus management and keyboard interaction support

## Props

| Prop               | Type        | Default     | Description                                      |
| ------------------ | ----------- | ----------- | ------------------------------------------------ |
| `isOpen`           | `boolean`   | `false`     | Controls the visibility of the modal             |
| `onClose`          | `function`  | `required`  | Function called when modal should be closed      |
| `title`            | `string`    | `undefined` | Optional title displayed at the top of the modal |
| `children`         | `ReactNode` | `required`  | Main content of the modal                        |
| `footer`           | `ReactNode` | `undefined` | Optional footer content (buttons, actions, etc.) |
| `className`        | `string`    | `''`        | Additional CSS classes for the modal container   |
| `overlayClassName` | `string`    | `''`        | Additional CSS classes for the backdrop overlay  |

## Usage Examples

### Basic Modal

```jsx
const [isOpen, setIsOpen] = useState(false);

<Modal isOpen={isOpen} onClose={() => setIsOpen(false)} title="Welcome">
  <p>This is a simple modal with just a title and content.</p>
</Modal>;
```

## Accessibility Considerations

- The modal traps focus within its boundaries
- Pressing the Escape key should close the modal (can be implemented in the parent component)
- The modal is announced to screen readers with proper ARIA attributes
- Background content is not accessible while modal is open

## Best Practices

1. **Always provide an `onClose` function** to handle modal closing
2. **Use descriptive titles** to help users understand the modal's purpose
3. **Keep content concise** to avoid overwhelming users
4. **Include clear action buttons** in the footer for user interactions
5. **Test keyboard navigation** to ensure accessibility compliance
