A flexible chips component perfect for displaying tags, labels, categories, or any removable content pieces. This component supports multiple variants, sizes, deletion functionality, clickable links, and custom styling. Ideal for filter interfaces, tag systems, user selections, or any scenario where you need compact, interactive content badges.

## Features

- **Multiple Variants**: Different visual styles (default, primary, secondary, success, warning, danger)
- **Size Options**: Small, medium, and large size variants
- **Deletable**: Optional close button with custom delete handlers
- **Clickable Links**: Can function as navigation links or clickable elements
- **Icons Support**: Add leading or trailing icons to chips
- **Custom Styling**: Complete control with Tailwind CSS classes
- **Accessibility**: Proper ARIA attributes and keyboard navigation
- **Animation**: Smooth hover and deletion transitions

## Props

| Prop        | Type                                                                          | Default     | Description                                   |
| ----------- | ----------------------------------------------------------------------------- | ----------- | --------------------------------------------- |
| `label`     | `string`                                                                      | `required`  | The text content of the chip                  |
| `variant`   | `'default' \| 'primary' \| 'secondary' \| 'success' \| 'warning' \| 'danger'` | `'default'` | Visual variant of the chip                    |
| `size`      | `'sm' \| 'md' \| 'lg'`                                                        | `'md'`      | Size of the chip                              |
| `deletable` | `boolean`                                                                     | `false`     | Whether the chip can be deleted               |
| `onDelete`  | `function`                                                                    | `undefined` | Function called when delete button is clicked |
| `onClick`   | `function`                                                                    | `undefined` | Function called when chip is clicked          |
| `href`      | `string`                                                                      | `undefined` | URL for link chips                            |
| `icon`      | `ReactNode`                                                                   | `undefined` | Leading icon element                          |
| `endIcon`   | `ReactNode`                                                                   | `undefined` | Trailing icon element                         |
| `className` | `string`                                                                      | `''`        | Additional CSS classes                        |
| `disabled`  | `boolean`                                                                     | `false`     | Whether the chip is disabled                  |

### Deletable Chips

```jsx
function TagManager() {
  const [tags, setTags] = useState([
    { id: 1, label: "Frontend" },
    { id: 2, label: "Backend" },
    { id: 3, label: "Database" },
    { id: 4, label: "DevOps" },
  ]);

  const handleDelete = (tagId) => {
    setTags(tags.filter((tag) => tag.id !== tagId));
  };

  return (
    <div className="flex gap-2 flex-wrap">
      {tags.map((tag) => (
        <Chip
          key={tag.id}
          label={tag.label}
          deletable
          onDelete={() => handleDelete(tag.id)}
          variant="primary"
        />
      ))}
    </div>
  );
}
```

### Status/Category Chips

```jsx
function StatusChips({ items }) {
  const getVariant = (status) => {
    switch (status) {
      case "completed":
        return "success";
      case "in-progress":
        return "warning";
      case "pending":
        return "secondary";
      case "error":
        return "danger";
      default:
        return "default";
    }
  };

  return (
    <div className="space-y-2">
      {items.map((item) => (
        <div
          key={item.id}
          className="flex items-center justify-between p-3 border rounded"
        >
          <span>{item.name}</span>
          <Chip
            label={item.status}
            variant={getVariant(item.status)}
            size="sm"
          />
        </div>
      ))}
    </div>
  );
}
```

### Filter Chips with Search

```jsx
function FilterableChips() {
  const [selectedFilters, setSelectedFilters] = useState([]);
  const [searchTerm, setSearchTerm] = useState("");

  const availableFilters = [
    "JavaScript",
    "React",
    "Vue",
    "Angular",
    "Node.js",
    "Python",
    "Django",
    "Flask",
    "Express",
    "MongoDB",
  ];

  const filteredOptions = availableFilters.filter(
    (filter) =>
      filter.toLowerCase().includes(searchTerm.toLowerCase()) &&
      !selectedFilters.includes(filter)
  );

  const addFilter = (filter) => {
    setSelectedFilters([...selectedFilters, filter]);
    setSearchTerm("");
  };

  const removeFilter = (filter) => {
    setSelectedFilters(selectedFilters.filter((f) => f !== filter));
  };

  return (
    <div className="space-y-4">
      <div>
        <input
          type="text"
          placeholder="Search filters..."
          value={searchTerm}
          onChange={(e) => setSearchTerm(e.target.value)}
          className="w-full px-3 py-2 border rounded focus:outline-none focus:ring-2 focus:ring-blue-500"
        />
      </div>

      {/* Selected filters */}
      {selectedFilters.length > 0 && (
        <div>
          <h4 className="text-sm font-medium mb-2">Active Filters:</h4>
          <div className="flex gap-2 flex-wrap">
            {selectedFilters.map((filter) => (
              <Chip
                key={filter}
                label={filter}
                variant="primary"
                deletable
                onDelete={() => removeFilter(filter)}
              />
            ))}
          </div>
        </div>
      )}

      {/* Available filters */}
      {filteredOptions.length > 0 && (
        <div>
          <h4 className="text-sm font-medium mb-2">Available Filters:</h4>
          <div className="flex gap-2 flex-wrap">
            {filteredOptions.map((filter) => (
              <Chip
                key={filter}
                label={filter}
                variant="secondary"
                onClick={() => addFilter(filter)}
              />
            ))}
          </div>
        </div>
      )}
    </div>
  );
}
```

## Variant Styles

### Visual Variants

**Default**

- Light gray background with dark text
- Subtle and neutral appearance

**Primary**

- Blue color scheme
- Used for important or primary actions

**Secondary**

- Purple/indigo color scheme
- For secondary actions or categories

**Success**

- Green color scheme
- Indicates positive status or completion

**Warning**

- Yellow/orange color scheme
- For cautions or pending states

**Danger**

- Red color scheme
- For errors or delete actions

## Best Practices

1. **Use consistent variants** - Stick to a variant system across your application
2. **Provide clear labels** - Use descriptive, concise text that users can understand
3. **Group related chips** - Organize chips logically when displaying multiple items
4. **Handle deletion gracefully** - Provide confirmation for destructive actions when necessary
5. **Limit chip length** - Keep labels short to maintain clean layouts
6. **Consider mobile experience** - Ensure chips are appropriately sized for touch interaction
7. **Use appropriate colors** - Match chip variants to your application's color scheme and meaning
