A versatile dropdown component that provides a clean interface for selecting from a list of options. This component supports search/filter functionality, keyboard navigation, custom option rendering, and multiple selection modes. Perfect for forms, navigation menus, or any interface that requires option selection with an improved user experience.

## Features

- **Search/Filter**: Built-in search functionality to filter options
- **Keyboard Navigation**: Full keyboard support with arrow keys and Enter
- **Custom Option Rendering**: Support for complex option content with icons, descriptions
- **Multiple Selection**: Optional multi-select mode with checkboxes
- **Click Outside to Close**: Automatically closes when clicking outside the dropdown
- **Loading States**: Support for async options loading
- **Disabled States**: Individual options or entire dropdown can be disabled
- **Custom Styling**: Complete control over appearance with Tailwind CSS

## Props

| Prop              | Type                                                        | Default              | Description                                 |
| ----------------- | ----------------------------------------------------------- | -------------------- | ------------------------------------------- |
| `options`         | `Array<{value: string, label: string, disabled?: boolean}>` | `required`           | Array of dropdown options                   |
| `value`           | `string \| string[]`                                        | `''`                 | Selected value(s)                           |
| `onChange`        | `function`                                                  | `required`           | Function called when selection changes      |
| `placeholder`     | `string`                                                    | `'Select an option'` | Placeholder text when no option is selected |
| `searchable`      | `boolean`                                                   | `false`              | Enable search/filter functionality          |
| `multiple`        | `boolean`                                                   | `false`              | Enable multiple selection mode              |
| `disabled`        | `boolean`                                                   | `false`              | Disable the entire dropdown                 |
| `loading`         | `boolean`                                                   | `false`              | Show loading state                          |
| `className`       | `string`                                                    | `''`                 | Additional CSS classes for the container    |
| `optionClassName` | `string`                                                    | `''`                 | Additional CSS classes for option items     |
| `maxHeight`       | `string`                                                    | `'200px'`            | Maximum height of the dropdown list         |

## Keyboard Navigation

The dropdown supports full keyboard navigation:

- **Tab**: Focus the dropdown trigger
- **Enter/Space**: Open/close the dropdown
- **Arrow Up/Down**: Navigate through options
- **Enter**: Select the focused option
- **Escape**: Close the dropdown
- **Type**: Search for options (when searchable is enabled)

## Advanced Features

### Async Options Loading

```jsx
function AsyncDropdown() {
  const [options, setOptions] = useState([]);
  const [loading, setLoading] = useState(false);
  const [searchTerm, setSearchTerm] = useState("");

  const loadOptions = async (search = "") => {
    setLoading(true);
    try {
      const response = await fetch(`/api/users?search=${search}`);
      const users = await response.json();
      setOptions(
        users.map((user) => ({
          value: user.id,
          label: `${user.name} (${user.email})`,
        }))
      );
    } finally {
      setLoading(false);
    }
  };

  useEffect(() => {
    loadOptions(searchTerm);
  }, [searchTerm]);

  return (
    <Dropdown
      options={options}
      loading={loading}
      onSearch={setSearchTerm}
      searchable
      placeholder="Search users..."
    />
  );
}
```

## Best Practices

1. **Provide clear labels** - Use descriptive option labels that are easy to understand
2. **Limit option count** - Consider pagination or virtualization for large datasets
3. **Enable search for long lists** - Makes finding options much easier for users
4. **Handle loading states** - Show appropriate feedback during async operations
5. **Use appropriate selection modes** - Single vs multi-select based on use case
6. **Test keyboard navigation** - Ensure all functionality works without a mouse
7. **Consider mobile experience** - Ensure dropdown works well on touch devices
